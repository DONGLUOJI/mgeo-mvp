# 董逻辑 MGEO 检测系统技术文档

更新时间：2026-04-04

## 1. 项目概述

本项目是一个面向品牌 AI 搜索可见性检测的最小可运行系统，围绕 MGEO 方法构建，当前已经具备以下核心能力：

- 免费检测入口
- 多模型检测执行
- TCA 评分生成
- 报告展示与导出
- 检测历史管理
- 客户列表与任务列表
- SQLite / Postgres 双数据库支持
- Mock / 真实 API 混合运行模式

项目目标不是只做单次检测，而是为后续的诊断、服务转化、交付执行、效果监测和复盘迭代提供基础技术底座。

## 2. 当前技术栈

- 前端框架：Next.js 15
- 语言：TypeScript
- 路由模式：App Router
- 样式：全局 CSS
- 本地数据库：SQLite
- 线上数据库：Postgres
- 模型接入方式：多 provider 适配层
- 报告导出：HTML 模板 + 浏览器打印导出 PDF

## 5. 检测主流程

### 5.1 用户输入

检测页当前接收的字段包括：

- `brandName`
- `industry`
- `businessSummary`
- `query`
- `selectedModels`

### 5.2 提交流程

`/api/detect` 处理逻辑如下：

1. 校验输入字段
2. 生成 `taskId`
3. 组装 `DetectInput`
4. 判断所选模型是否已配置真实 provider
5. 若有真实 provider，则调用 `runDetect`
6. 若没有真实 provider，则回退到 `buildMockReport`
7. 保存 mock 兼容数据
8. 保存数据库记录
9. 返回 `taskId` 和评分信息

对应核心文件：

## 12. 当前运行模式

当前系统支持三种检测运行状态：

### 12.1 Mock 模式

适用于：

- 未配置真实模型 API
- 演示环境

特点：

- 全流程可跑
- 结果固定可控
- 方便前端调试

### 12.2 Real 模式

适用于：

- 所选模型都已配置真实 API

特点：

- 所有结果来自真实模型返回

### 12.3 Hybrid 模式

适用于：

- 部分模型已配置真实 API
- 部分模型未配置

特点：

- 已配置的模型走真实 API
- 未配置的模型自动回退 mock

这是当前最推荐的开发模式。
